> **This is a pre-release** (`6.3.0-prerelease-rcN`). It has been validated
> in staging and is undergoing SQA testing. Production deployments should
> await the stable release.

## Highlights

- **ConfigMap-based configuration** — Service configs (pools, backends, roles, templates) can now be managed as Helm values via Kubernetes ConfigMap, enabling GitOps workflows with ArgoCD.
- **Unified gateway chart** — Envoy, OAuth2 Proxy, and the authz sidecar deploy as a single gateway component. The standalone router chart is merged into the service chart.
- **Rsync download support** — Pull files from running workflow tasks to your local machine with `osmo workflow rsync download`, with visual progress bars for both uploads and downloads.
- **Workflow submit performance** — Group and task database inserts are now batched atomically, reducing query count from O(N) to constant time.
- **Python 3.14 and Pydantic v2** — The platform migrates to Python 3.14 and Pydantic v2, improving validation performance and enabling modern Python features.

## Breaking Changes

- **Router chart merged into service chart**: The standalone `router` Helm chart no longer exists. Router pods now deploy as part of the `service` chart. Run `helm uninstall router` before upgrading, then upgrade the service chart. See the [upgrade guide](https://github.com/NVIDIA/OSMO/blob/main/deployments/upgrades/6_2_to_6_3_upgrade.md) for migration steps. (#897)
- **ConfigMap mode blocks CLI config writes**: When `services.configs.enabled: true`, CLI and API config updates return HTTP 409. Update your Helm values instead. (#822)

## Configuration Management

- **ConfigMap-sourced configuration**: All service configs (pools, backends, pod templates, group templates, resource validations, roles, dataset buckets) can be managed via a Kubernetes ConfigMap mounted from Helm values. The service loads configs at startup and watches for file changes. (#822)
- **File-backed authz sidecar**: The authorization sidecar reads roles directly from the ConfigMap file, eliminating its PostgreSQL dependency for role resolution. (#822)
- **Startup validation with K8s Events**: Invalid ConfigMap values cause the pod to crash-loop at startup, preserving rolling-update protection. Hot-reload failures emit Warning events on the ConfigMap visible via `kubectl describe`. (#889)
- **Secret value redaction**: Pydantic validation errors no longer echo input values, preventing accidental credential exposure in logs and K8s Events. (#891)
- **Per-field Secret mounts**: Credentials can reference individual Secret keys instead of requiring a single `cred.yaml` file. (#884)
- **Computed field resolution at load time**: Pool `parsed_pod_template` and `parsed_resource_validations` are resolved from template name references when the ConfigMap loads, reducing config file size. (#866)
- **Config export script**: A migration script exports existing database configs to Helm values format for ConfigMap migration. (#866)

## Gateway and Routing

- **Unified gateway chart**: Envoy, OAuth2 Proxy, and the authz sidecar deploy together under `gateway.*` values. The gateway serves as the single LoadBalancer entrypoint for all services. (#799, #817)
- **Router merged into service chart**: Router pods deploy as part of the service chart release. Resource names are unchanged. (#897)
- **Default identity headers for minimal deployments**: New `gateway.envoy.defaultIdentity.*` values inject identity headers when OAuth2 Proxy and authz are disabled, allowing UI access without an IdP. (#902)
- **OAuth2 Proxy extraEnv support**: Environment variables can be passed to the oauth2-proxy container, enabling Redis AUTH configuration via Secret refs. (#898)
- **Gateway HPA support**: Horizontal pod autoscaling for the gateway deployment. (#811)

## CLI

- **Rsync download**: Pull files from running tasks to your local machine with `osmo workflow rsync download`. (#792)
- **Rsync progress bars**: Visual progress indicators show transfer status, percentage, rate, and ETA during rsync uploads and downloads. (#826)
- **Rsync foreground default**: Upload runs in foreground by default. Pass `--daemon` for background monitoring. (#792)
- **Workflow version navigation**: Navigate between workflow spec versions with `osmo workflow version`. (#834)
- **Token expiry warning**: CLI warns when access tokens are within 24 hours of expiry. (#711)
- **Uninstall script**: `osmo-uninstall` removes CLI binaries, shell completions, and config files. (#710)
- **Agent skill installation prompt**: macOS and Linux installers offer to install the OSMO agent skill for AI coding assistants. (#841)

## Web UI

- **Task failure messages**: Canceled tasks with null exit codes now display their failure message. Failed task details show both exit code and failure reason. (#832, #833)
- **Terminal resize**: Shell sessions dynamically resize to match the browser window, fixing vim rendering issues. (#717, #727)
- **Dataset manifest sort fix**: Binary search alignment corrected for dataset manifest sorting. (#903)

## Performance

- **Atomic group and task inserts**: Workflow submit batches group and task inserts in a single transaction. (#821)
- **Bulk workflow fetch**: `Workflow.fetch_from_db` loads all tasks in one query instead of per-group, reducing from 2N+2 queries to 3. (#820)
- **Batch group status updates**: `UpdateGroup` job uses batch updates and lightweight aggregation queries. (#742, #769)
- **Barrier notification batching**: `_notify_barrier` uses a single SQL query and Redis pipeline instead of N individual calls. (#756)
- **Concurrent workflow file uploads**: `UploadWorkflowFiles` uses async concurrent uploads. (#783)
- **Kombu queue priority**: Background jobs support priority ordering. (#773)

## Security

- **Privilege escalation fix**: Policies with empty resources lists no longer incorrectly grant access to resource-scoped endpoints. (#867)
- **Credential name masking fix**: Credential field names (access_key_id, secret_access_key) are no longer masked in workflow specs when they appear as identifiers rather than secret values. (#744)
- **Credential env var collision fix**: Multiple credentials sharing payload key names no longer overwrite each other. Keys are namespaced with the credential name. (#839)
- **Dependency security updates**: grpc-go 1.80.0, pgx 5.9.1, docker 28.5.2, cryptography 46.0.7, flatted 3.4.2. (#869)

## Backend Operator

- **Squid proxy removed**: Backend operator no longer requires the squid proxy for egress. Network policy restricts pod-to-pod traffic instead. (#823)
- **Shared memory pod template**: Default `shared_memory` template mounts a 64Gi memory-backed emptyDir at `/dev/shm`. (#837)

## Platform

- **Python 3.14**: All services migrate from Python 3.10 to 3.14. (#840)
- **Pydantic v2**: Models migrate to Pydantic v2 with improved validation and performance. (#828)
- **PyJWT 2.12.0**: JWT library updated. (#752)

## Bug Fixes

- **Authz path fixes**: Corrected authorization paths for rsync, workflow exec, and credential creation. (#737, #738, #739)
- **Default pool submission**: Users with `osmo-user` role can submit to the default pool without specifying it. (#728)
- **Workflow restart access**: Fixed API access for workflow restart operations. (#716)
- **User fetch API**: Corrected user fetch endpoint behavior. (#753)
- **macOS installer directory creation**: Ensures `/usr/local/bin` exists before copying binaries. (#714)
