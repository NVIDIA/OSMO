> **This is a pre-release** (`6.3.0-prerelease-rc1`). It has been validated in
> staging and is undergoing SQA testing. Production deployments should await
> the stable release.

## Highlights

- **ConfigMap-sourced configuration** — Manage OSMO service configuration via Helm values and Kubernetes ConfigMaps, enabling GitOps workflows with ArgoCD and version-controlled configuration.
- **Unified gateway architecture** — Consolidated Envoy, authz sidecar, and OAuth2 proxy into a single gateway chart, simplifying deployment and reducing resource overhead.
- **Python 3.14 and Pydantic v2** — Upgraded runtime to Python 3.14 and migrated to Pydantic v2 for improved performance and modern language features.
- **Rsync bidirectional file transfer** — CLI now supports downloading files from workflow tasks with visual progress bars, complementing existing upload functionality.
- **Improved workflow performance** — Batch database operations reduce query count from O(N) to O(1) for workflow submissions and group updates.

## Breaking Changes

- **Chart consolidation**: The standalone Router chart has been merged into the Service chart. Deployments using the Router chart separately must migrate to `services.router` within the Service chart values. (#897)
- **ConfigMap mode writes blocked**: When `services.configs.enabled: true`, API and CLI writes to configuration endpoints return 409 Conflict. Update Helm values instead. (#822)
- **Gateway as load balancer**: All service types now route through the unified gateway. Review `gateway.*` values if using custom Envoy or OAuth2 proxy configurations. (#817)

## Authentication and Authorization

- **Default identity headers**: Minimal deployments without OAuth2 or authz can now configure default identity headers (`gateway.envoy.defaultIdentity`) for UI functionality. (#902)
- **Privilege escalation fix**: Policies with empty resources lists no longer incorrectly grant access to resource-scoped endpoints. (#867)
- **OAuth2 proxy extraEnv**: The oauth2-proxy container now supports `extraEnv` for injecting secrets like Redis passwords. (#898)
- **Default role sync**: Users receive the default role prior to IDP role synchronization. (#791)

## Workflow Engine

- **Atomic batch inserts**: Group and task database inserts are now batched into a single transaction, preventing orphan groups on partial failures. (#821)
- **Bulk workflow loading**: `Workflow.fetch_from_db()` reduced from 2N+2 queries to 3 queries regardless of group count. (#820)
- **Barrier notification optimization**: Redis pipeline batches set, expire, and per-task lpush into a single round trip. (#756)
- **Concurrent file uploads**: Workflow file uploads now use async concurrent transfers for faster submission. (#783)
- **Job queue priority**: Kombu queue now supports message priority for critical jobs. (#773)
- **Workflow version navigation**: Users can navigate between workflow versions in the UI. (#834)
- **Credential collision fix**: Multiple credentials using the same payload key name no longer overwrite each other. (#839)

## Data and Storage

- **Rsync download**: Pull files from remote workflow tasks to local machines via `osmo rsync download`. (#792)
- **Rsync progress bars**: Visual progress display with transfer rate and ETA for both uploads and downloads. (#826)
- **Credential name masking fix**: Credential names and field references in workflow specs are no longer incorrectly redacted. (#744)

## Scheduling and Compute

- **Shared memory pod template**: Default `shared_memory` pod template mounts a memory-backed volume at `/dev/shm` for ML frameworks requiring shared memory. (#837)
- **Pool computed field resolution**: ConfigMap pools now resolve pod templates and resource validations by reference, reducing config file size by ~63%. (#866)
- **Custom HPA metrics**: Operators can specify custom Horizontal Pod Autoscaler metrics for service scaling. (#858)

## Web UI

- **Failure messages for canceled tasks**: Tasks canceled with null exit codes now display their failure message in the details panel. (#833, #832)
- **Dynamic terminal sizing**: Vim and other terminal applications render correctly with dynamic window resize support. (#717, #727)
- **Filter and retry fixes**: Resolved issues with workflow filters, task retries, and log viewing. (#784)

## CLI

- **Rsync subcommands**: Restructured rsync CLI with `upload`, `download`, `status`, and `stop` subcommands. Upload now runs in foreground by default; use `--daemon` for background mode. (#792)
- **Token expiry warning**: CLI displays warnings when access tokens are approaching expiration. (#711)
- **Agent skill prompt**: Installer prompts to install the OSMO agent skill for AI coding assistants. (#841)
- **Uninstall script**: New `uninstall.sh` script for clean CLI removal. (#710)

## Bug Fixes

- **Secret value redaction**: ConfigMap validation errors no longer echo plaintext secrets in logs or K8s Events. (#891)
- **ConfigMap fail-fast**: Malformed ConfigMaps at startup now cause pod crash-loops instead of silent fallback to stale database config. K8s Events are emitted on hot-reload failures. (#889)
- **Stream error handling**: 403 errors in workflow file streaming are properly caught and returned as 500 responses. (#730)
- **Manifest sort alignment**: Dataset manifest comparator now aligns with binary search expectations. (#903)
- **Fetch user API**: Fixed user fetch endpoint returning incorrect data. (#753)
- **Restart API access**: Corrected authorization for workflow restart endpoints. (#716)
