## Highlights

- **ConfigMap-based configuration** — Manage all service configs (pools, backends, templates, roles) as Helm values via a Kubernetes ConfigMap instead of the database. Follows standard K8s patterns and supports GitOps workflows with ArgoCD.
- **Router chart merged into service chart** — The standalone router Helm chart is gone; router pods now deploy as part of the service chart, simplifying deployment to a single Helm release.
- **Rsync download and progress bars** — Pull files from running tasks to your local machine and see real-time transfer progress during uploads and downloads.
- **Faster workflow submission** — Batched database operations reduce queries from 2N+2 to 3 when loading workflows with many groups and tasks.
- **Privilege escalation fix** — Policies with empty resources lists no longer grant access to resource-scoped endpoints.
- **Secret redaction in validation errors** — ConfigMap validation errors no longer leak credential values into logs or K8s Events.

## Breaking Changes

- **Router chart removed**: The standalone `osmo/router` chart no longer exists. Router pods now deploy as part of the service chart. Uninstall any existing router release before upgrading. See the [6.2 to 6.3 upgrade guide](https://nvidia.github.io/OSMO/deployment_guide/) for cutover steps. (#897)
- **Squid proxy removed from backend operator**: The backend operator no longer deploys a squid proxy. Workflow pods now egress directly to configured destinations. Network policies restrict cross-pod communication within the cluster. (#823)

## Helm Charts

- **ConfigMap-sourced configuration**: Service configs can be managed as Helm values via `services.configs`. When enabled, the service loads pools, backends, templates, roles, and credentials from a mounted ConfigMap. CLI/API config writes return HTTP 409. (#822)
- **Fail-fast startup on bad ConfigMap**: Malformed ConfigMap files cause pods to crash-loop at startup instead of silently falling back to stale database configs. K8s Events are emitted on hot-reload failures for operator visibility. (#889)
- **Per-field Secret mounts**: Secrets can be created with `kubectl --from-literal` instead of a single `cred.yaml` file. The loader reads each file in the mount directory as a credential field. (#884)
- **Pool computed fields resolved at load time**: ConfigMap pools no longer require pre-expanded `parsed_pod_template` and `parsed_resource_validations`, reducing config file size by ~63%. (#866)
- **Default identity headers for minimal deployments**: Gateway can inject default user, roles, and allowed-pools headers when no identity provider is configured, making the UI usable in demo mode. (#902)
- **Extra environment variables for oauth2-proxy**: Pass `OAUTH2_PROXY_*` env vars from Secret refs, needed when Redis session store requires authentication. (#898)
- **Custom HPA metrics**: Specify custom Horizontal Pod Autoscaler metrics for all service components. (#858)
- **Gateway chart consolidated**: Envoy, authz-sidecar, and oauth2-proxy resources moved into the service chart's gateway section. (#799, #817)

## CLI

- **Rsync download**: Pull files or directories from running tasks to your local machine with `osmo workflow rsync download`. (#792)
- **Rsync progress bars**: Foreground transfers show real-time progress with bytes transferred, percentage, rate, and ETA. (#826)
- **Uninstall script**: Remove OSMO CLI with `osmo uninstall` or the standalone `uninstall.sh` script. (#710)
- **Token expiry warning**: CLI warns when your access token is within 24 hours of expiry. (#711)
- **Agent skill installation prompt**: macOS and Linux installers offer to install the OSMO agent skill for AI coding assistants. (#841)
- **Vim rendering fix**: Terminal resizes correctly when using vim inside `osmo workflow exec`. (#717)

## Web UI

- **Workflow version navigation**: Navigate between workflow versions from the details panel header. (#834)
- **Task failure messages**: Failed and canceled tasks show their failure message in the Details section, even when the exit code is null. (#832, #833)
- **Shell window resizing**: The exec shell respects terminal size changes. (#727)

## Performance

- **Batched workflow submission**: Groups and tasks insert atomically in a single transaction. (#821)
- **Bulk workflow loading**: Fetching a workflow with G groups now takes 3 queries instead of 2G+2. (#820)
- **Batched group status updates**: Status updates batch multiple groups in one query instead of per-group round trips. (#769, #742)
- **Batched barrier notifications**: Redis pipeline batches set/expire/lpush calls for barrier members. (#756)
- **Concurrent workflow file uploads**: Workflow files upload concurrently instead of sequentially. (#783)

## Security

- **Privilege escalation fix**: Policies with an empty resources list only match unscoped paths, not resource-scoped endpoints. Previously, users with `auth:Token` and no resources could manage other users' tokens. (#867)
- **Secret redaction in validation errors**: Pydantic validation errors no longer echo credential values into logs or K8s Events. Only field paths and types are shown. (#891)
- **Credential names preserved in workflow specs**: Credential field references (`access_key_id`, `secret_access_key`) are no longer incorrectly masked by the secret redaction filter. (#744)
- **Credential env var collision fix**: Multiple credentials with the same payload key name no longer overwrite each other. Keys are namespaced with the credential name. (#839)

## Bug Fixes

- **Dataset manifest sort comparator**: Manifest binary search now uses a comparator that matches the sort order. (#903)
- **Default pool submission**: Users with the `osmo-user` role can submit to the default pool without specifying it explicitly. (#728)
