## Workflow

The primary entity users interact with. Represents a job submitted to run on compute resources.

### Actions

| Action | CLI Command | Description |
|--------|-------------|-------------|
| Submit | `workflow submit` | Submit workflow from file or rerun |
| List | `workflow list` | Filter by user, pool, status, date |
| Get | `workflow get` | Get details, verbose mode |
| Cancel | `workflow cancel` | Cancel running workflow |
| Delete | `workflow delete` | Delete completed/failed workflow |
| Restart | `workflow restart` | Restart a workflow |
| Logs | `workflow logs` | Stream task logs |
| Shell | `workflow shell` | Interactive shell into task |
| Exec | `workflow exec` | Execute command in task |
| Port Forward | `workflow port-forward` | Forward ports to task |
| Rsync | (via submit --rsync) | Sync files to running task |
| Spec | `workflow spec` | View workflow spec |

### Statuses

```
WAITING, PROCESSING, SCHEDULING, INITIALIZING, RUNNING, COMPLETED, FAILED,
FAILED_EXEC_TIMEOUT, FAILED_START_ERROR, FAILED_START_TIMEOUT, FAILED_SERVER_ERROR,
FAILED_BACKEND_ERROR, FAILED_QUEUE_TIMEOUT, FAILED_IMAGE_PULL, FAILED_UPSTREAM,
FAILED_EVICTED, FAILED_PREEMPTED, FAILED_CANCELED
```

### Priority Levels

| Priority | Description |
|----------|-------------|
| HIGH | Scheduled first, never preempted |
| NORMAL | Default priority |
| LOW | May be preempted for higher priority |
